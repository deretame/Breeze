# Rust 侧错误消息国际化（fluent-rs）

`rquickjs_playground` 与 `rust`（windcore）两个 crate 使用 [fluent-rs](https://github.com/projectfluent/fluent-rs) 维护 Rust 侧错误/日志消息的多语言切换。默认语言为 `zh-CN`，Dart 侧调用 `setQjsErrorMessageLanguage(lang: "en-US")` 切换。

## 翻译文件位置

```text
rquickjs_playground/locales/zh-CN/messages.ftl
rquickjs_playground/locales/en-US/messages.ftl
```

FTL 文件即翻译的唯一来源，新增/修改文案时直接编辑这两个文件，保持中英文消息 ID 一致。

## 添加一条新消息

1. 在两个 FTL 文件中写入相同的消息 ID：

   ```ftl
   # zh-CN/messages.ftl
   failed-to-do-something = 做某事失败: { $name }

   # en-US/messages.ftl
   failed-to-do-something = Failed to do something: { $name }
   ```

2. 在 Rust 代码中使用：

   ```rust
   rquickjs_playground::tr!("failed-to-do-something", name = value)
   ```

   在 `rquickjs_playground` crate 内部可用 `crate::tr!(...)`；在 `windcore` 中用 `rquickjs_playground::tr!(...)`。

## API 速查

```rust
// 无参
rquickjs_playground::tr!("message-id")

// 带具名参数
rquickjs_playground::tr!("message-id", url = bundle_url, status = status_code)

// 函数形式
rquickjs_playground::i18n::tr("message-id");
rquickjs_playground::i18n::tr_args("message-id", &[("url", &bundle_url)]);

// 切换语言
rquickjs_playground::i18n::set_locale("en-US");
```

## 注意事项

- Fluent 消息 ID 必须以字母开头，可包含字母、数字、`-`、`_`。
- 占位符统一使用 `{ $name }` 具名变量，不再使用 `{0}`、`{1}` 位置参数。
- `tr!` 返回 `String`。在需要 `&str` 的上下文（如 `.contains()`、`.expect()`）中，使用 `&tr!("...")`。
- 修改 FTL 后运行 `cargo check -p rquickjs_playground`，若 FTL 语法有误会在编译期报错。
